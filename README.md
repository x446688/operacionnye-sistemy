# opercaionnye-sistemy
Добро пожаловать в корень репозитория проектов по "Операционным системам" и смежным дисциплинам! 

В данном проекте содержатся разборы практических заданий, курсовые работы, а также лекционные практики по следующим правилам:
1. В директории `works` можно найти практические работы, в том числе `mysyslog` и `myRPC`. 
2. В директории `lecutres` можно найти программы из лекционных примеров.
3. Директория `kursach-2025` является сабмодулем ([**Тык**](https://github.com/x446688/curse-ovaya-rabota-OS-2025)) 

## Немного о верзионировании

Позволю себе уделить один абзац подробному описанию моей философии верзиониварония. Изначально мною рассматривалась стандартная схема выдачи версии для каждого подпроекта индивидуально, что было бы логично, если бы всё это не лежало в одном репозитории по ТЗ. Возможно, есть альтернативные и более эффективные способы разбора данной ситуации, но я действую по следующей схеме:
1. Понять, что конкретно затронул новый релиз / хотфикс.
2. Если не затрагиваются системные компоненты, то прибавить буквенный суффикс к версии и тегировать релиз.
3. Иначе инкрементировать версию в соответствии с масштабом изменений.

Наглядно:
```
# не затронули релиз программы вообще
1.2.1b
| | ||
| | |\---- не затронута системная логика программы
| | \----- хотфикс / незначительное количество изменений
| \------- добавлено значительное количество изменений
\--------- программа чуть ли не полностью переработана
```

> Версии с добавычными суффиксами не будут учтены как полноценный релиз.  
## c2rn 1.0
В [курсовой работе](https://github.com/x446688/curse-ovaya-rabota-OS-2025) с тематическим названием `c2rn` акцент делается на механизмы сборки (и тестирования) при помощи GNU Autotools и check. Собрано произольное приложение, разработанное на C с использованием GTK и OpenSSL. Более подробно о приложении можно узнать, ознакомившись с его исходными кодами.
## mysyslog 1.0
В рамках данной директории реализована библиотека libmysyslog, а также два драйвера для вывода текста в разных форматах. Данная библиотека создаёт удобный интерфейс для записи файлов в системный журнал с выбранным уровнем логировани и форматом. Дополнительно вместе с библиотекой реализована программная обёртка над библиотекой - добавлен клиент mysyslog-client, позволяющий тестировать библиотеку с помощью изменения набора передаваемых аргументов. Для более подробной информации лучше ознакомиться с [README.md в директории](works/mysyslog/README.md).
## myRPC 1.0
В рамках данной директории реализован механизм удаленного вызова команд myRPC, который использует сокеты в качестве транспорта и JSON формат сообщений. В директории находятся два подроекта - клиент и сервер. В рамках сервера реализована функция демонизации, аналогичная mysyslog, но с незначительными изменениями. Для более подробной информации лучше ознакомиться с [README.md и requirements.md в директории](work/myRPC/README.md).
