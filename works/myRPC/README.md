# myRPC
## Описание проекта
myRPC - это механизм удаленного вызова команд, который использует сокеты в качестве транспорта и протокол json для передачи команд. Механизм состоит из двух элементов: утилиты myRPC-client и демона myRPC-server. 
## Общая схема работы
#### myRPC-client
1. Пользователь при помощи аргументов командной строки в утилите myRCP-client задает параметры подключения к серверу, тип сокета и команду на языке BASH.
2. Утилита получает имя пользователя, от имени которого она запущена, подключается через механизм сокетов к серверу
3. Если она смогла успешно подключиться к серверу передает ему в формате JSON имя пользователя и команду.
#### myRPC-server
1. Во время запуска получает из конфигурационного файла /etc/myRPC/ myRPC.conf номер порта и тип сокета (потоковый или датаграмный)
2. Получает через механизм сокетов строку в формате JSON с именем пользователя и командой
3. Разбирает строку, получает из нее имя пользователя и команду
4. Проверяет в конфигурационном файле /etc/myRPC/users.conf, есть ли пользователь в списке допущенных пользователей
5. Если пользователь есть в списке выполняет команду, записывая весь вывод stdout во временный файл по шаблону /tmp/myRPC_ХХХХХХ.stdout, а весь вывод ошибок во временный файл по шаблону /tmp/myRPC_ХХХХХХ.stderr. 
## Аргументы командной строки myRPC-client:
`-c / --command` - Команда BASH  
`-h / --host` - Адреса сервера  
`-p / --port` - Порт сервера  
`-s / --stream` - Тип потокового сокета  
`-d / --dgram` - Тип датаграмного сокета  
`--help` - Вывод помощи  

## Пример содержимого конфигурационного файла
```conf
# CONF START
port = 1234
socket_type = stream
# CONF END
```
> В системе пользователь myrpc должен обладать всеми соответствующими правами на файлы. (см. [`myRPC.service`](esrver/myRPC.service))
