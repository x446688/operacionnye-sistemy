# myRPC-server (systemd)

## Что я устанавливаю?
`myRPC-server` - это программа-демон (в зависимости от параметров запуска), выполняющая функцию сервера получателя-обработчика удаленного вызова команд. Сервер поддерживает json протокол вызова команд.
- Формат запроса на выполнение команды:
```json
{“login”:”имя_пользователя”,”command”:”команда bash”}
```
- Формат ответа сервера:
```json
{“code”:код_ответ, ”result”:”результат исполнения или описание ошибки”}
```
Код_ответа – 0, в случае успеха, в поле result записываются данные stdout  
Код_ответа – 1, в случае ошибки, в поле result записываются данные stderr  

Все логи программы записываются в заранее установленный файл логов с помощью libmysyslog.

## Установка
```sh
make deb
dpkg -i myRPC-server.deb
```
Этот набор команд установит:
1.  `myRPC-server` в `/usr/bin/`
2. `myRPC.conf` в `/etc/myRPC/`  
3. `users.conf` в `/etc/myRPC/`  
4. `myRPC-server.service` в `/usr/lib/systemd/system/`
## Использование

Для проверки `myRPC-server` без демонизированного режима достаточно запустить программу с соотстветствующими правами на файлы логов и конфигурации.

```sh
myRPC-server
```

Если установлен сервис и правильно настроены права, то его можно запустить с помощью команд (вторая команды из цепочки проверяет состояние сервиса):

```sh
sudo systemctl start myRPC-server.service
sudo systemctl status myRPC-server.service
```

Ознакомиться со списком доступных опций для сервера можно передав соответственно аргумент `-h` или `--help` исполняемому файлу.

`--help / -h` - напечатает полный список опций   
`--conf-file / -c filename` - прочитать конфигурацию из `filename`  
`--log-file / -l filename` - записывать логи в `filename`  
`--daemon / -d` - демонизировать приложение  
`--pid-file / -p filename` - установить PID-file для сервиса  
`--xtreme / -x` - запустить xtreme режим: принудительно возвращать `stdout` в форму возврата JSON в поле `result`
